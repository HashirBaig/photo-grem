Short Description

We take 3 overlapping aerial survey images into ERDAS IMAGINE. Using this tool, we generate Tie Points, perform Aerial Triangulation and then generate 2 DSMs from the two overlapping regions. These 2 DSMs are then merged together in CloudCompare to produce a final DSM as a LAS file which is then rasterized. A similar process is performed with the 2 DSM LAS files acquired from the AHN website to produce a single LAS file as well as a rasterized file. From both these LAS files, we also generate DTMs as raster files using the Cloth Simulation Filter. Next, we add checkpoints to compare the DSM and the DTM generated by our team against those from the AHN website. Based on the checkpoints, we then calculate the overall RMSE as well as the Uncertainty per checkpoint for both the DSM and DTM. The output rasters are of 1 meter resolution.

Our website relies on Firebase as the backend service for storing most of our geospatial data through a storage bucket (hosted by Google Cloud Platform). Further, we rely on Netlify for hosting the website. The website is rendered using HTML, CSS, and JavaScript. Firstly, we add a basemap from OSM using Leaflet. For rendering the rasters we use GeoRaster, while we render the vector files using Leaflet. Additionally, the map controls are also created using Leafelt. Additionally, we use Potree to provide a 3D visualization of the DSM generated by our team. The rendered 3D representation of the DSM (using potree) is stored on the ITC server due to space constraints.
