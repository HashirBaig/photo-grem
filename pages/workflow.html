FAQs for Photogrammetric Product

Were the UAV images used to generate the photogrammetric products georeferenced?
Yes, they have already been georeferenced. The provided block file contains the orientation parameters.
What parameters were used while running the Cloth Simulation Filter for generating the Ground and Off-Ground points?
The "scene" parameter was set to flat since the terrain in The Netherlands is generally flat. The other parameters were kept default.
How were interpolation distances for the rasterization of the photogrammetric products (DSM & DTM) chosen?
The interpolation was performed using the least distance possible such that there would be no holes in the raster (DSM and DTM) after the rasterization process.
How were the checkpoints for estimating the RMSE values for the DSM and the DTM chosen?
Spatial spread of the checkpoints is the main criteria while choosing. For the DSM, the chosen checkpoints lie on roads, cycle lanes, and building tops, while for the DTMs we chose only roads and cycle lanes. We avoided croplands and forested areas due to their seasonal growth variability.
What is the reasoning behind representing Uncertainty using circles?
In our work, uncertainty is calculated as the difference in elevations between our photogrammetric products and the AHN reference data. Since this is an absolute value, it can be represented adequately using Size as a graphic variable. Hence, the uncertainty is represented using the diameter of the circle.
What are the sources of uncertainty in these Photogrammetric Products?
Major sources of uncertainty include:
a. Uncertainty due to improper camera calibration of UAV in the z-direction.
b. Uncertainty during Bundle Adjustment in ERDAS.
c. Uncertainty in the Cloth Simulation Filter (CSF) operation, which assumes a certain type of terrain.
d. Uncertainty in the Interpolation process during rasterization of DSM & DTM.

FAQs for Website

Where is the data stored?
The point cloud is stored in ITC servers. DSM, DTM, and related files are stored in Firebase.
Why did we use Firebase?
Firebase is capable of storing and syncing data in real-time across all devices.
What open standards and web services are used?
Storage buckets in Firebase and OSM maps from Leaflet.
What tools were used for photogrammetry?
Leaflet for 2D data, Potree for 3D point cloud rendering.
What tools were used in website design?
Figma for UI design, HTML, CSS, JS, and Bootstrap for development.

About Our Web App

Welcome to our cutting-edge web application, designed to revolutionize how photogrammetric products are analyzed and visualized. Our platform utilizes the powerful Cloud Compare software to create DSM, DTM and Orthophoto from aerial imagery and AHN lidar data points, enabling users to gain precise insights into surface and urban changes.

What We Offer

DSM and DTM creation

Leveraging ERDAS and Cloud Compare's capabilities, our platform delivers highly accurate DSM and DTM results, making it an essential tool for professionals in urban plannin and disaster assessment.

Orthophoto creation

A true Orthophoto is created from the drone images. Our platform produces high-resolution, geometrically corrected orthophotos, ensuring accurate mapping and analysis for applications in land surveying, agriculture, and infrastructure planning.

Web-Based Accessibility

No software installation required. Our web app allows users to access and visualise data from anywhere, ensuring convenience and efficiency.
